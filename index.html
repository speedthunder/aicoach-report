安裝與啟動：就像開啟 App 一樣簡單

只需要有電腦、有網路，在終端機輸入一行指令就能開始。不需要懂複雜的程式語法，用中文打字就能操作。

2

AI Agent 是什麼？就是「自動完成任務的 AI 機器人」

普通 AI 只會「回答問題」，但 AI Agent 會「採取行動」。它能主動規劃步驟、呼叫工具、執行程式，像一個真正的員工一樣替你工作，完成後再報告給你。

# 範例：讓 Claude Code 幫你建立 AI 健康管理 Agent 你說： "幫我建立一個 AI Agent， 能自動讀取病患生理數據， 並在異常時發送 LINE 通知" # Claude Code 會自動： # 1. 分析需求 # 2. 規劃架構 # 3. 寫出完整程式碼 # 4. 測試並修正錯誤

精準健康管理社區據點

在高齡化社會浪潮下，社區居民的健康管理需求日益迫切。這個競賽題目要我們用最前端的 Agentic AI 技術，打造一個真正能主動關心住戶、即時響應健康警報的智慧社區系統——這不只是比賽，更是一次改變臺灣醫療照護模式的機會。

🔍 一、問題背景與商機

台灣正快速步入超高齡社會。根據國發會預測，2026 年老年人口比率將突破 20%。然而，現有的社區健康服務仍停留在「被動等待」模式——住戶有不舒服才去看診，沒有預防性的主動關懷機制。

現況痛點 1：數據孤島

生理量測數據分散在各設備，無法整合分析，健康趨勢不可見

現況痛點 2：服務被動

藥師、營養師只能等住戶上門，無法主動發現高風險族群

商機 1：精準健康管理

數據驅動的個人化健康建議，比傳統衛教更有效率

商機 2：社區服務升級

AI 將被動服務人員升級為主動式「健康教練」，提升服務價值

💡 二、核心解決方案：Agentic AI 智慧健康平台

我們要建立一個「會主動思考、主動行動」的 AI 系統，不是只會傳送固定通知的機器人。

①
多源數據採集

整合式生理量測儀（血壓、血氧、心率）+ 居家穿戴裝置 + 社區 App，持續累積睡眠、活動量、互動行為等多維數據

②
AI 智慧分析引擎

Agentic AI 平台整合近期生理數據、設備使用頻率與歷史趨勢，自動判斷健康風險等級，識別異常行為模式

③
主動個人化互動

AI Agent 不發固定通知，而是自動生成個人化關懷訊息，透過 App / LINE 推送適合的健康建議與社區服務推薦

④
雙層觸發機制

住戶未回應時，自動升級觸發第二層：通知社區服務人員（藥師/營養師）主動聯繫，形成人機協作的照護網絡

⑤
閉環學習優化

互動結果、參與狀況持續回寫平台，作為 Agent 後續判斷依據，系統越用越聰明，服務越來越精準
